## 3. Annotation Behavior

### 3.1 Overview

Annotations allow structured commentary on specific portions of recipe text. They are stored as repeater rows on the Recipe CPT and displayed via JavaScript interaction.

### 3.2 Anchor Matching

Annotations are linked to text via **exact string matching** of the `anchor_text` field.


If the anchor appears multiple times, the `occurrence` field specifies which instance (1-indexed).

### 3.3 Display States

**Default state:**
- Anchored text is highlighted (subtle underline or background)
- Notes panel shows placeholder: "Click any highlighted term for commentary."

**Hover state:**
- Anchor text changes color or gains emphasis
- Cursor changes to pointer

**Active state (clicked):**
- Anchor text is visually selected (stronger highlight)
- Notes panel slides in with annotation content
- Other annotations remain available via clickable anchors

## 4. Filter Logic

### 4.1 Recipe Archive Filters

| Filter | Source | Query Type | Logic |
|--------|--------|------------|-------|
| Source (Work) | Work CPT (type: ancient_text) | meta_query | Recipes where source_work.parent_work OR source_work matches |
| Period | Period taxonomy | tax_query | Standard taxonomy filter |
| Ingredient | Ingredient Product CPT | Complex | See Â§4.2 |
| Process | Process CPT | meta_query | Recipes linked to this process |

### 4.2 Ingredient Filter Logic

Filtering recipes by ingredient requires traversing the interpretation chain:

1. User selects: "Myrrh resin" (Ingredient Product)
2. Query finds: All Identifications where `ingredient_product` = Myrrh resin
3. Query extracts: Ancient Ingredient IDs from those Identifications
4. Query finds: All Recipes where `ingredients` repeater contains any of those Ancient Ingredient IDs

```php
function aos_filter_recipes_by_product($product_id) {
    // Get all identifications pointing to this product
    $identifications = get_posts([
        'post_type' => 'identification',
        'meta_query' => [
            ['key' => 'ingredient_product', 'value' => $product_id]
        ],
        'fields' => 'ids'
    ]);
    
    // Extract ancient ingredient IDs
    $ancient_ids = [];
    foreach ($identifications as $id) {
        $ancient_id = get_field('ancient_ingredient', $id);
        if ($ancient_id) $ancient_ids[] = $ancient_id;
    }
    
    // Find recipes using these ancient ingredients
    // (requires custom query on repeater data)
    return aos_get_recipes_with_ingredients($ancient_ids);
}
```

### 4.3 Filter UI Behavior

**Single-select dropdowns (MVP):**
- Each filter is independent (OR logic between filters would require v1.1)
- Selecting a filter value reloads the page with URL parameter
- URL format: `/recipes/?source=de-materia-medica&period=roman`

**Filter counts:**
- Display count of matching recipes next to each filter option
- e.g., "Myrrh resin (12)"

**Clear filters:**
- Single button to reset all filters
- Labeled "Clear filters" or "Show all"

### 4.4 URL Parameters

| Parameter | Values | Example |
|-----------|--------|---------|
| `source` | Work slug | `?source=de-materia-medica` |
| `period` | Period term slug | `?period=roman` |
